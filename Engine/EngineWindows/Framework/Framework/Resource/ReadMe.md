框架库模块

框架库（Framework）位于引擎的最上层，封装了与操作系统相关的基础框架（包括Windows、Linux、Android和Macintosh），但不直接调用系统API。
并对引擎游戏中间层进行基础设计，实现各个系统的分离。
框架库允许调用引擎中的所有库，并使用boost工具库。

----------------------------
1.	Helper（帮助）

框架库帮助模块。
（1）用户宏：用于判断是否使用静态库及其他用户设定。 
（2）主函数宏入口点：各个系统的main函数的入口点函数的封装。
（3）主函数宏：各个系统的main函数调用的宏定义。
（4）中间层宏：引擎中间层获取管理器指针函数的宏定义。

----------------------------
2.	MainFunctionHelper（主函数帮助）

主函数帮助类。封装了控制台、Windows、OpenGL、Android、Macintosh等操作系统的入口函数，使得不同平台都能正常运行。

（1）环境目录：环境目录的设置和获取，读取文件的路径默认不增加环境目录，需要调用者通过环境目录修改文件名。
（2）主函数帮助基类：外部类控制单例的初始化和销毁。内部类目前只保存环境目录，待后续功能添加框架所需的帮助信息。
（3）控制台主函数帮助：控制台主函数帮助类，获取命令行参数对象，以实现各个平台的控制台入口点。
（4）Windows主函数帮助：Windows主函数帮助，实现Windows平台的入口点以及其他平台的虚拟入口。
（5）OpenGL GLUT主函数帮助：OpenGL GLUT主函数帮助，实现各个平台的OpenGL GLUT控制台入口点。
（6）Android主函数帮助：Android主函数帮助，实现Android平台的入口点以及其他平台的虚拟入口。
（7）Macintosh主函数帮助：Macintosh主函数帮助，实现Macintosh平台的入口点以及其他平台的虚拟入口。

----------------------------
3.	WindowCreate（窗口创建）

窗口创建相关类。主要实现Windows平台的功能，包括窗口点、大小、创建参数、实例参数、句柄等，并使其他操作系统也能正常调用这些接口。

（1）窗口大小：窗口大小类，并定义了相关操作运算符。
（2）窗口点：窗口点类，并定义了相关操作运算符。
（3）窗口参数：创建窗口所需要参数的封装，以简化函数传递。
（4）窗口句柄：窗口句柄类，封装Windows平台的HWND，使其他操作系统也能正常调用。

----------------------------
4.	WindowProcess（窗口进程）

Windows窗口进程相关类。主要实现虚拟键类型、进程管理、进程句柄、消息循环和消息接口的基类。

（1）虚拟键类型：支持Windows平台和GLUT的虚拟按键。
（2）消息循环：消息循环类，封装Windows平台的主消息循环入口。
（3）消息接口：Windows平台消息接口的基类和带中间层参数的消息类。
（4）进程管理：Windows平台的进程管理，将Windows平台的回调函数转发给消息接口类。
（5）进程句柄：Windows平台的进程句柄，连接进程管理单例和注册类的通信。
（6）Windows消息单元测试套件：Windows消息单元测试套件，测试需要Windows平台窗口环境的单元测试。

----------------------------
5.	WindowRegister（窗口注册）

Windows的窗口注册相关类。包括窗口注册需要的各种参数，以及光标、图标、背景刷等。

（1）注册参数：Windows平台注册窗口所需参数。
（2）注册图标类：Windows平台注册窗口所需光标、图标、背景刷等。
（3）注册名字：Windows平台注册窗口所需名字和菜单名字。
（4）注册句柄：Windows平台注册句柄，连接框架和窗口进程的通信。

----------------------------
6.	Application（应用）

封装了各个系统的键盘和鼠标的控制，各平台的应用基础信息和像素场景、摄像机运动、对象运动等应用相关的类。

（1）键盘和鼠标的控制：各平台的键盘和鼠标的控制的接口，用于监听键盘和鼠标消息。
（2）字体信息：各平台的字体信息，以获得字符串的高度和宽度。
（3）性能测量：程序的性能测量，并将信息打印到控制台。
（4）应用基础信息：各平台的应用基础信息，由主函数生成。
（5）像素场景：一个像素场景类，对屏幕每个点的颜色进行显示和修改，并通过渲染器实现绘图功能。
（6）摄像机运动：一个摄像机运动类，通过键盘和鼠标修改摄像机位置。
（7）对象运动：一个对象运动，通过键盘和鼠标修改对象位置。
（8）引擎测试名字：一个测试名字的集合，用于获取某个索引的测试名字。

----------------------------
7.	MiddleLayer（中间层）

封装了各个中间层类，包括模型、视图、控制器系统、含像素绘制的中间层、摄像机和对象运动的中间层和一个游戏系统管理器结构。
这个游戏系统管理器分为输入管理器、网络管理器、对象逻辑、物理模拟、消息管理器、事件管理器、系统管理器、资源管理器、音频管理器、相机系统、渲染管理器、GUI管理器构成。

（1）模型、视图、控制器系统：实现模型、视图、控制器系统。
（2）游戏系统管理器系统：实现游戏系统管理器系统。

封装了Windows API框架。使得其他系统也能正常调用Windows框架。

（1）Windows API框架：实现Windows API框架，实现窗口注册、窗口进程、窗口创建的功能。

封装了OpenGL GLUT API框架。使得各种系统能正常调用OpenGL API框架。还包括回调、进程等相关类。

（1）OpenGL GLUT回调：实现OpenGL GLUT API回调消息接口类，将消息转发给通过消息接口。
（2）OpenGL GLUT进程：实现OpenGL GLUT API进程相关类，连接进程管理单例和框架类的通信，将回调函数转发给消息接口类。
（3）OpenGL GLUT API框架：实现OpenGL GLUT API框架，实现窗口进程的功能。
