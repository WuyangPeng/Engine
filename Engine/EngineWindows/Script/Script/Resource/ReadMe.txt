脚本库模块

1.	Macro（宏）

脚本库模块系统宏。
目前包含：
（1）用户宏：用于是否使用静态库、是否使用lua等用户设定。
 待实现功能：
（1）脚本库宏：脚本库模块所使用的宏，按需要添加。

2.	Configuration （配置）

封装了脚本库的配置。
 待实现功能：
（1）配置策略：对脚本库的相关策略进行封装，按需要后续添加。 
（2）配置解析：对脚本库配置进行解析，返回配置的策略。

3.	Interface（接口）

封装了脚本库的统一接口。 处理模式使用策略类，分别代表使用哪种脚本库。 
内部使用Java、lua、Perl、Python、或者TCREScript。接口尽量保持同Python一致。 
 待实现功能：
（1）调用：实现各脚本语言的统一调用接口。

4.	Java（Java）

封装了调用java代码的实现。
 待实现功能：
（1）调用：实现java的调用接口。

5.	Lua（lua）

封装了调用lua代码的实现。
 待实现功能：
（1）调用：实现lua的调用接口。

6.	Perl（Perl）

封装了调用perl代码的实现。
 待实现功能：
（1）调用：实现perl的调用接口。

7.	Python（Python）

封装了调用Python代码的实现。
 待实现功能：
（1）调用：实现Python的调用接口。

8.	TCREScript（TCREScript） 

封装了引擎设计的一种脚本语言。
 待实现功能：
（1）调用：实现TCRE脚本的调用接口。
