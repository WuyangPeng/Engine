网络库模块

1.	Helper（帮助）

封装了网络库所需要的宏。
目前包含：
（1）输入输出流宏：网络库解析消息时所需要的宏。
（2）用户宏：用于是否使用静态库、是否使用ACE、openssl等用户设定。

2.	Configuration（网络库配置）

封装了网络库的配置。包括，包装器外观、组件配置器、截取器和扩展接口。
目前包含：
（1）配置策略：对服务器和客户端的相关策略进行封装，按需要后续添加。
（2）配置子策略：对服务器和客户端的相关子策略进行封装，按需要后续添加。
（3）配置参数：对服务器和客户端的相关配置参数进行封装，按需要后续添加。
（4）配置解析：对网络库配置进行解析，返回配置的策略。

3.	NetworkMessage（网络消息）

封装了网络消息的实现。
目前包含：
（1）消息接口：设计消息类的接口。
（2）消息事件：设计消息事件和回调事件。
（3）消息类型：封装消息类型。
（4）消息流：封装消息流，以实现消息的读取和发送。
（5）套接字管理：套接字管理，以封装网络套接字。
待实现功能：
（1）protobuf：使用protobuf来解析消息。
（2）json：使用json来解析消息。

4.	Interface（网络库接口）

封装了网络库的统一接口。处理模式使用策略类，分别代表反应器、主动器、异步完成标记、接受器――连接器以及其他并发模式。
对框架类，有一个Run()函数调用，可以通过策略类实现创建线程、回调函数等功能。对面向链接使用连接者、接受者和流三个角色对象。
内部使用CoorTools、boost::asio、boost::beast 或者 ACE。接口尽量保持同boost::asio一致。 

目前包含：
（1）基础主管理：封装各种实现所需要的特殊函数。
（2）网络地址：封装各种实现的地址类。
（3）接受器：封装各种实现的接受器类。
（4）连接器：封装各种实现的连接器类。
（5）流：封装各种实现的流类。
（6）句柄集：封装ACE实现所需要的句柄集，其他实现为空实现。
（7）发送套接字：封装各种实现的发送套接字类。
（8）客户端：封装一个客户端的基础实现。
（9）服务器：封装一个服务器的基础实现。

5.	ACEWrappers（ACE包装器）

封装了ACE的实现。
目前包含（部分功能未实现）：

（2）网络地址：封装ACE的地址类。
（3）接受器：封装ACE的接受器类。
（4）连接器：封装ACE的连接器类。
（5）流：封装ACE的流类。
（6）句柄集：封装ACE实现所需要的句柄集。

6.	BoostWrappers（boost包装器）

封装了boost的实现。
目前包含：
（1）基础主管理：封装boost所需要的特殊函数和线程管理。
（2）网络地址：封装boost的地址类。
（3）接受器：封装boost的接受器类，可选择同步阻塞、同步非阻塞或异步的实现。
（4）连接器：封装boost的连接器类，可选择同步、或异步的实现。
（5）流：封装boost的流类，可选择同步阻塞、同步非阻塞或异步的实现。
